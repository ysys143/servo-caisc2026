# Frozen R24 coding manual

## Unit and boundary

The unit is a source-defined system version, not a unique research lineage. The fourteen records represent thirteen lineages because the AI Scientist v2 preprint and the 2026 record belong to one lineage. Code only what the frozen packet reports. Absence of reporting is not evidence of absence.

A validator channel is an identifiable evaluator with a stable target, substrate, decision role, or feedback path. Split channels when one named system uses distinct mechanisms for ideation filtering, search control, terminal assessment, or external acceptance.

## Facets

- `operational_status`: distinguish implemented behavior, unexercised proposals, external processes, non-reporting, and genuinely unclear descriptions.
- `target_property`: the property judged, such as executability, correctness, novelty, or aggregate quality.
- `evidence_source`: the information used to judge the target.
- `evaluator_substrate`: the mechanism or actor performing the judgment.
- `decision_role`: the operational consequence of the output.
- `feedback_path`: where the output is actually routed. A terminal assessment is not search control.
- `external_independence`: organizational or informational independence of the evaluator.
- `reliability_evidence_type`: the study design reported for the exact channel and target. Lack of documented bias is `none_reported`, not calibration.
- `reliability_finding`: the conclusion supported by that evidence.
- `experimental_fidelity`: the environment in which the evaluated evidence is produced.

Every channel requires at least one exact packet quote. Never transfer evidence from a terminal paper reviewer to an ideation novelty filter merely because both use an LLM.

## Closure and human authority

Closure is derived rather than freely rated. Implemented policy control or stage transition is closed computational feedback; memory update or candidate revision alone is partial; terminal-only or external-only assessment is open. Unclear routing remains unclear.

Human authority is recorded separately for generator, executor, validator, memory, and policy. Use only observable roles: `explicit_none`, `initiate`, `advise`, `select`, `approve`, `execute`, `final_authority`, `not_reported`, or `not_applicable`.

## Missingness

`not_reported` means the packet provides no claim. Explicit none means the source states absence. `unclear` means relevant text exists but supports multiple readings. `not_applicable` means the facet has no coherent value for that channel. These values are never interchangeable.
